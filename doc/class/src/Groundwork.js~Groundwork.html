    Groundwork

    For backwards compatability apiKey is aliased to oauth_client_id within the config dictionary. The clientId getter/setters remain next to the same getters/setters for apiKey. This allows code written for older versions of groundwork.js to continue to use oauth_client_id to configure an instance while new code is encouraged to use apiKey.

    Example:

    let gw = new Groundwork({'apiKey': '1234'});
