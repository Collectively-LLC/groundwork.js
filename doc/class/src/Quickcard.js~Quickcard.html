    public create(quickcard: Object): Promise source

    POST a quickcard record to the API

    User must be authed / have a gwid

    The passed in form object will be validated. If it fails, a mock response with any errors will be sent back in a rejected Promise. This is to present the least amount of surprise to the developer.

    Note: All of the values in the object passed to the method should have had their types coerced already or validation will fail.

    Params:

    NameTypeAttributeDescription
    quickcard Object
    • optional

    Return:

    Promise

    Example:

    // validation fail
donation.create({}).catch((err) => console.log(err));
// => { status: 400, data: { error: { valid: false, ... } } ... }

    public del(id: String, time: ...time<number>): Promise source

    Delete (cancel) a quickcard

    Note: Quickcards can be cancelled on a date in the future by passing additional arguments.

    Params:

    NameTypeAttributeDescription
    id String

    quickcard id
    time ...time<number>
    • optional

    year, month, day for a specific epoch

    Return:

    Promise

    Example:

    // Cancel quickcard abc123 today
del('abc123');

// Set a cancellation date of Oct 13, 2015 for quickcard abc123
del('abc123', 2015, 10, 13)
